# Прототип UI

Це **прототип візуальної частини** застосунку — без реальної логіки, з фейк-даними.
Мета: оцінити дизайн, навігацію, форми **перш ніж писати бекенд**.

## Як відкрити

Двічі клікнути на `index.html` — відкриється у браузері за замовчуванням.
Між сторінками переходи через sidebar зліва і кнопки.

## Сторінки

1. **index.html** — Dashboard (головна, з KPI і останніми варками)
2. **bulk-list.html** — Журнал варок (список з фільтрами)
3. **bulk-new.html** — Нова варка (форма з нормами зі специфікації)
4. **spec-card.html** — Картка специфікації RP032
5. **passport.html** — Картка паспорта якості з превʼю PDF

## Що працює / не працює

✅ Працює:
- Навігація між сторінками через sidebar
- Розгортання/згортання груп у sidebar (Pico деталі)
- Responsive layout (можна звузити вікно)

❌ Не працює (бо це прототип):
- Збереження даних
- Кнопки "Зберегти", "Завантажити PDF", "Створити нову версію"
- Фільтри (відображаються, не фільтрують)
- Перерахунок висновку OK/OUT у формі варки

## Стек прототипа

- HTML5 (один файл = одна сторінка)
- [Pico.css v2](https://picocss.com/) з CDN — semantic CSS framework
- Власний `styles.css` (~250 рядків) — лише sidebar layout і дрібні віджети

## Як змінити дизайн

Усі стилі sidebar, badge'і, "норма-факт" поля — у `styles.css`.
Контент сторінок — у відповідних `.html` файлах.

Sidebar дублюється в кожному файлі (для статичного прототипа це норма).
У реальному застосунку буде один `app/templates/base.html` через Jinja2.
